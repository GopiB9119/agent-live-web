# Daily Memory Log - 2026-03-03

## Context
- User requested reliability improvements after repeated live-web drift and chat-evaluation inconsistencies.
- Focus shifted from individual site actions to systemic prompt/instruction hardening.

## Issues Observed
- Overlay and onboarding states blocked actions and caused false retries.
- Generic/global selectors drifted to hidden or stale elements.
- Chat success was sometimes marked without validating a new assistant response.
- Broad text extraction occasionally surfaced stale content instead of latest assistant message.
- Diagnostics were polluted by extension-origin warnings mixed with first-party runtime signals.

## Changes Made
- Added mandatory dynamic UI/chat controls in `.github/agents/agent-live-web.agent.md`:
  - H1 pre-action UI gate
  - H2 visible-only scoped selectors
  - H3 two-phase chat verification
  - H4 latest-response extraction rule
  - H5 blocked-action recovery ladder
  - H6 diagnostics source filtering
  - H7 anti-drift completion guard
- Propagated equivalent controls into:
  - `.github/copilot-instructions.md`
  - `.github/instructions/playwright-edge.instructions.md`
- Added public project note in `README.md` under `Reliability standards (web + chatbot)`.

## Decision
- Reliability controls must live in both agent prompt and repository instructions so behavior remains stable even if execution mode/prompt changes.

## Expected Effect
- Fewer false-positive passes in chat tasks.
- Better resilience to dynamic overlays and modal gates.
- Cleaner triage by separating first-party errors from tooling noise.
